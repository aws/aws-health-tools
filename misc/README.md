# S3 Cross-Region Copy Tools

Amazon S3 is designed for 99.999999999% (11 nines) durability and 99.99% availability. In the vast majority of cases, your data remains fully accessible. However, there can be instances of prolonged degraded availability in a region — such as the incident starting March 1, 2026 affecting the `me-central-1` and `me-south-1` regions. S3 recommends enabling [Cross-Region Replication (CRR)](https://docs.aws.amazon.com/AmazonS3/latest/userguide/replication.html) to maintain a live replica of your data in a different region for multi-region redundancy.

When an incident does occur and you need to copy data out of an affected region, this folder contains automation scripts for the recommended approaches, ordered from most to least managed.

## Quick Decision Guide

**If S3 control plane + Batch Operations with replication are available in the source region** → Use [CRR + Batch Replication](#option-1-crr--batch-replication) — it's the best managed solution and preserves all object properties.

**If only the S3 data plane is available in the source region** → Use [AWS DataSync](#option-2-aws-datasync) — runs from the destination region and does not require a manifest to be provided.

**If all objects are < 5 GB and you need Batch Operations control** → Use [S3 Batch Operations Copy](#option-3-s3-batch-operations-copy) — runs from the destination region but requires a manifest.

**If you have very large buckets (> 20 billion objects)** → Use [S3 Inventory to Batch Replication](#option-4-s3-inventory-to-batch-replication) — splits inventory across multiple batch replication jobs.

**If you have a small set of objects** → Use the AWS CLI [`aws s3 sync`](https://docs.aws.amazon.com/cli/latest/reference/s3/sync.html) or [`aws s3 cp`](https://docs.aws.amazon.com/cli/latest/reference/s3/cp.html) commands to copy data directly. Note that tags and metadata are silently dropped for objects > 5 GB with `s3 sync`.

## Option 1: CRR + Batch Replication

📁 [`S3-replication-script/`](S3-replication-script/)

Sets up S3 Cross-Region Replication (CRR) on the source bucket and uses S3 Batch Replication to bootstrap existing objects. New objects are replicated automatically going forward.

**Scripts:**
- `setupReplication-me-central-1.py` — One-shot script that creates the replication IAM role, destination bucket, enables versioning, configures CRR, and submits a batch replication job. Supports `--monitor` for polling completion.

**Dependencies:**
- S3 control plane available in source region
- S3 Batch Operations available in source region

**How copied objects look:** All properties fully preserved — tags, user metadata, ACLs, version IDs, object creation time, storage class, and encryption.

## Option 2: AWS DataSync

📁 [`s3-cross-region-copy-using-aws-datasync/`](s3-cross-region-copy-using-aws-datasync/)

Uses AWS DataSync (Enhanced mode) to copy objects between S3 buckets. Tasks are created and run from the **destination region**, minimizing source region dependency.

**Scripts:**
- `create_datasync_task.py` — Creates IAM roles, DataSync locations, and tasks. Supports CSV batch processing for multiple buckets, test mode with include filters, and auto destination bucket creation.
- `check_task_status.py` — Monitors task execution with `--monitor` flag. Outputs JSON/CSV.
- `cleanup_datasync_tasks.py` — Removes DataSync resources (dry-run by default, never deletes buckets).

**Dependencies:**
- S3 data plane available in source region (DataSync reads via S3 API)
- DataSync service available in destination region

**How copied objects look:** Tags and user metadata are preserved. Version IDs, object creation time, and ACLs are **not** preserved (new objects created at destination). Storage class is configurable. Enhanced mode copies only the latest version of the object by default. All versions can be copied by providing a manifest.

## Option 3: S3 Batch Operations Copy

📁 [`s3-cross-region-copy-using-batch-operations/`](s3-cross-region-copy-using-batch-operations/)

Runs S3 Batch Operations copy jobs from the **destination region**. Requires generating a manifest of objects to copy.

**Scripts:**
- `generate_manifest.py` — Lists objects in the source bucket and produces CSV manifests. Splits objects into ≤ 5 GB and > 5 GB manifests. Supports versioned buckets with `--include-versions`.
- `create_batch_copy_jobs.py` — Creates S3 Batch Operations copy jobs with auto IAM role and destination bucket creation. Jobs created in Suspended state by default.
- `copy_large_objects.py` — Multipart copy for objects > 5 GB, preserving ETags. Dry-run by default.

**Dependencies:**
- S3 data plane available in source region
- Manual manifest generation (ListObjectsV2 or S3 Inventory)
- S3 Batch Operations available in destination region

**How copied objects look:** Tags and user metadata are preserved. Version IDs and object creation time are **not** preserved. Storage class is configurable.

**⚠️ Limitations:** KMS-encrypted objects (SSE-KMS) are not supported by these scripts. Native Batch Operations copy is limited to objects ≤ 5 GB; use `copy_large_objects.py` for larger objects.

## Option 4: S3 Inventory to Batch Replication

📁 [`S3-inventory-to-batch-job/`](S3-inventory-to-batch-job/)

For very large buckets (> 20 billion objects) that exceed the single Batch Replication job limit. Splits S3 Inventory CSV manifests across multiple concurrent Batch Replication jobs.

**CLI subcommands:**
- `setup-iam-role` — Creates IAM role with automatic KMS key detection.
- `setup-replication-rules` — Configures CRR rule on the source bucket.
- `split` — Partitions inventory manifest into sub-manifests. Supports dry-run and count-objects modes.
- `replicate` — Creates batch replication jobs (suspended state by default).
- `validate-setup` — Pre-flight checks for versioning, replication rules, IAM, inventory config, KMS, and report bucket.

Supports command chaining for one-shot execution.

**Dependencies:**
- S3 control plane + Batch Operations available in source region
- Pre-existing S3 Inventory (CSV format, `IncludedObjectVersions=All`)

**How copied objects look:** All properties fully preserved (same as Option 1).

## Prerequisites

All scripts require:
- Python 3.8+
- AWS credentials configured (`aws configure` or IAM role)
- `boto3` installed (`pip install boto3`)

See each tool's README for specific setup instructions.
